## Design Context

### Users
Researchers, analysts, developers, and knowledge workers who need research-grade search with source traceability. They use Perplexica in focused work sessions — often with multiple tabs, comparing sources, verifying claims. They value precision over flash.

### Brand Personality
Precise, authoritative, transparent — like a well-calibrated research instrument, not a chatbot.

### Aesthetic Direction
Linear 2D — flat, form-first, zero-depth. No shadows, no blur, no rounded pills. Borders define structure. Every surface is flush. Inputs are rectangular. Buttons are tight and bordered. Think: Linear.app issue detail, Notion database row, Vercel dashboard sidebar. Light mode default (research happens during work hours), dark mode for extended sessions.

**Visual system**: 4-8px radii, 1px borders, no box-shadow, no backdrop-blur, no gradients on surfaces. Accent color used sparingly on active states only. Monochrome palette with a single blue accent.

**References**: Linear (2D flat UI, border-first), Vercel Dashboard (clinical density), Bloomberg Terminal (data density done right)
**Anti-references**: ChatGPT (too conversational), generic AI chat UIs (glassmorphism, glowing accents, pill inputs)

### Design Principles
1. **Provenance is paramount** — Every claim traceable to its source, every output exportable with full metadata
2. **Accessibility is non-negotiable** — Research tools must work for everyone; WCAG AA minimum
3. **Data belongs to the user** — Export in every format, copy everything, own your research history
4. **Flat surfaces, sharp edges** — Borders not shadows, rectangles not pills, structure not decoration
5. **Density with clarity** — Show relevant metadata without clutter; progressive disclosure for depth
